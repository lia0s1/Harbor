# 性能执行边界

Harbor 的界面状态继续由主线程维护；可能涉及大量磁盘、JSON 或文本处理的工作不应占用该线程。

Xcode 的 Harbor 运行方案默认使用 Release，避免以未优化的 Debug 构建评估终端、监控和高频按钮交互；需要断点调试时再显式切换为 Debug。

## 已落实的路径

- 主机、快捷命令和脚本库的 JSON 编码与原子写盘在一个串行后台队列执行。正常退出时会等待该队列排空，保持原有的数据落盘保证。
- 主机备份导入/导出、`~/.ssh/config` 的读盘与解析在后台执行；合并到当前主机列表仍在主线程完成，避免覆盖导入期间的新改动。
- 远程目录列表的 UTF-8 解码、解析和排序在后台完成。主线程只接收最终条目，并继续通过代次检查丢弃过期结果。
- 内置编辑器读写临时文本文件时，最多 8 MB 的本地解码、编码和写盘均在后台进行；远程文件版本校验与冲突处理不变。
- 主机侧栏对会话状态只构建一次存活主机 ID 集合，行渲染不再为每台主机重复扫描全部会话。
- 主机侧栏的右键菜单、双击主操作及连接提示统一由列表容器承载，避免为每个主机行重复保留菜单、手势和提示元数据。

## 内存验收基线

以下数据用于观察大规模主机清单的空闲占用，不代表已连接终端、文件传输或监控面板的内存预算。

- 环境：Release 构建、Apple Silicon、macOS 26.4，隔离的应用支持目录，无打开会话，启动后稳定 15 秒。
- 负载：10,000 台 SSH 主机、每台两个标签（侧栏实际呈现约 20,000 行）、`hosts.json` 约 10.2 MB。
- 优化前（每行菜单、双击手势和提示）：物理内存 480.1 MB，峰值 598.1 MB。
- 优化后（列表级共享交互元数据）：物理内存 227.8 MB，峰值 344.1 MB；分别下降约 52.5% 和 42.5%。20 秒后的复测为 227.8 MB，未观察到空闲增长。
- `leaks` 对该隔离进程报告 0 leaks；受 macOS 进程附加限制，完整分配堆栈仍应在可调试构建的 Instruments 会话中采集。

## 现有边界

- 主机备份导入维持原有的 16 MB / 10,000 台主机上限。
- 内置编辑器维持原有的 8 MB 文本上限；更大的文件应使用外部编辑器。
- 远程命令、传输协议、认证和并发策略没有在本轮调整。

## 验证命令

```sh
xcodebuild -project Harbor.xcodeproj -scheme Harbor -configuration Debug \
  -destination 'platform=macOS' build CODE_SIGNING_ALLOWED=NO
swift test --package-path HarborKit
```
